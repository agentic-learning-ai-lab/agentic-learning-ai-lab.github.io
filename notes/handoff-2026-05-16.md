### Phase 4e — arxiv HTML inline figures via CDN

The "Full Paper (HTML)" view (when `enable_full_paper: true`) loads
`paper-content.json`'s embedded HTML through `paper-view.js`. The
HTML has `<img src="./assets/x1.png">` references that resolve
against `agenticlearning.ai/research/<slug>/assets/x1.png` (local
LFS, not CDN).

Migrating requires runtime path rewriting in `paper-view.js`:

```js
// Fetch the manifest once, then rewrite ./assets/<x> when injecting paper HTML
const manifest = await fetch('/assets-manifest.json').then(r => r.json());
function rewriteCdn(html, permalink) {
  return html.replace(
    /src="\.\/assets\/([^"]+)"/g,
    (m, rel) => {
      const url = manifest[`/research/${permalink}/assets/${rel}`];
      return url ? `src="${url}"` : m;
    }
  );
}
```

Effort ~1 hr including testing. Low urgency — the feature works
fine via LFS-served local paths today; CDN just makes it slightly
faster.

### WebP serving for hero / background images

PNG sources stay archival (in `assets/images/papers/`, `images/background/`,
`images/home/`); build emits WebP alongside via sharp, templates use
`<picture>` with PNG fallback.

Sketch:

```js
// build/generate_webp.js — runs after generate_thumbnails.js
const sharp = require('sharp');
const glob = require('glob');
const WEBP_PATHS = [
  'assets/images/papers/**/*.{png,jpg,jpeg}',
  'assets/images/background/**/*.{png,jpg,jpeg}',
  'assets/images/home/**/*.{png,jpg,jpeg}',
  'assets/images/people/**/*.{png,jpg,jpeg}',
];
// For each source, emit <basename>.webp at quality 85 next to it,
// skip if .webp newer than source. Add to SYNC_PATHS in sync_to_r2.js.
```

Template change — new helper `{{pictureCdn image}}` that returns:

```html
<picture>
  <source srcset="{{cdnUrl webp_variant}}" type="image/webp">
  <img src="{{cdnUrl image}}" ...>
</picture>
```

Where `webp_variant` is the same logical path with `.webp` extension. The
templater would pre-compute it (or the helper can derive it by swapping
extension). `assets-manifest.json` needs entries for both the PNG and the
.webp sibling.

Test plan: pick one paper hero, run end-to-end (build webp → sync → templater),
check `<picture>` renders with WebP in Chrome devtools and falls back to
PNG with WebP support disabled. Visual diff at quality 85 should be
indistinguishable for these use cases.

Touches: `build/generate_webp.js` (new), `build/sync_to_r2.js` (SYNC_PATHS),
`build/templater.js` (helper), `paper.hbs` + partials that currently use
`{{cdnUrl image}}`, package.json scripts (`build:webp`).

Effort ~1-2 hr. Storage cost on R2 is negligible (~80 MB extra at most);
real win is wire speed for first-paint.

## Important conventions saved this session

- **Never `git reset --hard`** in this repo. It wiped the
  `--latex-only` flag work mid-session. Use `git stash push -u`
  first, or `git rebase origin/main` instead. (Memory:
  `feedback_no_reset_hard.md`.)
- **Default to inline self-audit, not launched review agents.**
  Big launched-agent reviews take 60–150 sec each, too slow during
  iterative work. (Memory: `feedback_review_changes.md`.)
- **Run verification/data-gathering in the background** during
  interactive design conversations so foreground chat stays
  responsive. (Memory: `feedback_background_verification.md`.)
- **No Co-Authored-By lines** in commits. (Memory:
  `feedback_no_coauthor.md`.)
- **`lab-theme.css` and `includes/` are used externally** — don't
  delete or rename without checking other repos. (Memory:
  `project_shared_assets.md`.)
